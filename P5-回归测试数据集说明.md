# P5 回归测试数据集说明

版本：`v1.0`  
日期：`2026-03-30`

## 1. 目标

提供可重复执行的最小回归数据集，用于线路生成端到端回归测试。

覆盖要求：每个回归城市至少包含以下点位类型：

- `spot`
- `shopping`
- `restaurant`
- `hotel`
- `airport`

## 2. 已覆盖城市

- `CN / Beijing / Beijing`
- `MN / Ulaanbaatar / Ulaanbaatar`

每个城市当前均包含：

- `spot` x2
- `shopping` x2
- `restaurant` x2
- `hotel` x2
- `airport` x1 或以上

## 3. Seed 脚本

入口文件：

- `src/modules/trip-planner/seeds/run-regression-dataset-seed.ts`

执行命令：

```bash
npm run seed:regression
```

生产构建后执行：

```bash
npm run build
npm run seed:regression:prod
```

## 4. 数据规则

- 所有回归点位均设置为 `isPublished=true`。
- 所有回归点位均包含坐标、城市、省份、基础文案与建议停留时长。
- `restaurant` 均包含 `mealSlots`。
- `hotel` 均包含 `bookingUrl`。
- 机场使用 IATA 代码（如 `PEK/PKX/UBN`）。

## 5. 用途边界

- 本数据集用于回归与联调，不代表生产运营数据。
- 若需扩容城市，可按同样结构在 seed 文件追加城市定义。
