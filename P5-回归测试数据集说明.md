# P5 回归测试数据集说明

版本：`v1.1`  
日期：`2026-04-01`

## 1. 目标

提供可重复执行的最小回归数据集，用于线路生成端到端回归测试，并覆盖现实约束字段。

覆盖要求：每个回归城市至少包含以下点位类型：

- `spot`
- `shopping`
- `restaurant`
- `hotel`
- `airport`

## 2. 已覆盖城市

- `CN / Beijing / Beijing`
- `MN / Ulaanbaatar / Ulaanbaatar`

每个城市当前均包含：

- `spot` x2
- `shopping` x2
- `restaurant` x2
- `hotel` x2
- `airport` x1 或以上

并覆盖以下现实约束样例：

- 结构化营业时间
- 景点最晚入场时间
- 景点预约规则与预约时段
- 餐厅餐段时间窗
- 酒店入住退房时间
- 酒店可订状态与可订日期
- 机场到达/出发缓冲
- 路线锚点坐标

## 3. Seed 脚本

入口文件：

- `src/modules/trip-planner/seeds/run-regression-dataset-seed.ts`

执行命令：

```bash
npm run seed:regression
```

生产构建后执行：

```bash
npm run build
npm run seed:regression:prod
```

## 4. 数据规则

- 所有回归点位均设置为 `isPublished=true`。
- 所有回归点位均包含坐标、城市、省份、基础文案与建议停留时长。
- `spot` 均包含 `openingHoursJson + lastEntryTime`，部分景点包含预约规则。
- `shopping` 均包含锚点与结构化营业时间。
- `restaurant` 均包含 `mealSlots + mealTimeWindowsJson + 锚点`。
- `hotel` 均包含 `bookingUrl + foreignerFriendly + checkInTime/checkOutTime + bookingStatus + bookableDatesJson`。
- `airport` 使用 IATA 代码（如 `PEK/PKX/UBN`），并包含 terminal 锚点与缓冲时间。

## 5. 当前回归场景

- 北京：双机场、预约景点、午晚餐餐厅、双酒店、结构化营业时间。
- 乌兰巴托：单机场、城市景点、餐段餐厅、双酒店、结构化营业时间。

## 6. 用途边界

- 本数据集用于回归与联调，不代表生产运营数据。
- 若需扩容城市，可按同样结构在 seed 文件追加城市定义。
