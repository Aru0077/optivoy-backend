## 1. 范围

本文件冻结以下契约：

- backend `POST /trip-planner/generate` 请求/响应结构
- backend -> optimizer `/solve` 请求结构
- `transit_cache` 表结构（有向边模型）

不包含：

## 2. Backend 生成接口契约

接口：`POST /trip-planner/generate`

请求体（冻结）：

```json
{
  "city": "Beijing",
  "province": "Beijing",
  "arrivalDateTime": "2026-07-15T14:00:00+08:00",
  "arrivalBufferMinutes": 90,
  "selectedPointIds": ["uuid-1", "uuid-2"],
  "paceMode": "standard",
  "hotelMode": "multi",
  "mealPolicy": "auto",
  "arrivalAirportCode": "PEK",
  "departureAirportCode": "PEK"
```

字段说明：

- `selectedPointIds`：仅允许 `spot/shopping/restaurant`
- `paceMode`：`light|standard|compact`

- `mealPolicy`：`auto|off`
- `arrivalBufferMinutes`：`60-120`，默认 `90`

## 3. 到达缓冲时间规则

- 首日可用开始时间：`firstDayUsableStart = max(arrivalDateTime + arrivalBufferMinutes, paceStartTime)`
- 若 `firstDayUsableStart >= paceEndTime`：首日不排游玩点，仅安排入城/入住
- 缓冲时间不替代 `mealPolicy`，两者独立生效

接口：`POST /solve`

```json
{
  "city": "Beijing",

  "arrivalAirportCode": "PEK",

  "airportBufferMinutes": 90,
  "paceMode": "standard",

      "pointType": "spot",
      "suggestedDurationMinutes": 120,
      "latitude": 39.9,
      "longitude": 116.4
    }
  ],
  "hotels": [
    {
      "id": "hotel-uuid",
      "latitude": 39.9,
      "longitude": 116.4
    }
  ],
  "distanceMatrix": {
    "rows": []
  },
  "objective": "min_days",

```

说明：

- `distanceMatrix` 来自真实 `transit_cache` 数据
- 当矩阵缺边时，optimizer 会做坐标兜底估算

## 5. transit_cache 表结构（冻结）

本阶段采用**有向边**模型：`fromPointId -> toPointId`。

- 对同一城市，A->B 与 B->A 分开存储
- 唯一键：`(city, fromPointId, toPointId)`

SQL DDL（冻结）：

```sql
CREATE TABLE IF NOT EXISTS transit_cache (
  id uuid PRIMARY KEY DEFAULT uuid_generate_v4(),
  city varchar(120) NOT NULL,
  province varchar(120),
  "fromPointId" uuid NOT NULL,
  "fromPointType" varchar(16) NOT NULL,
  "toPointId" uuid NOT NULL,
  "toPointType" varchar(16) NOT NULL,
  "transitMinutes" int NOT NULL,
  "drivingMinutes" int NOT NULL,
  "walkingMeters" int NOT NULL,
  "transitSummary" text,
  "transitSummaryI18n" jsonb,
  "distanceKm" double precision NOT NULL,
  provider varchar(24) NOT NULL DEFAULT 'amap',
  status varchar(16) NOT NULL DEFAULT 'ready',
  "expiresAt" timestamp,
  "createdAt" timestamp NOT NULL DEFAULT now(),
  "updatedAt" timestamp NOT NULL DEFAULT now(),
  CONSTRAINT chk_transit_cache_from_to_not_equal CHECK ("fromPointId" <> "toPointId"),
  CONSTRAINT chk_transit_cache_from_point_type CHECK ("fromPointType" IN ('spot','shopping','restaurant','hotel','airport')),
  CONSTRAINT chk_transit_cache_to_point_type CHECK ("toPointType" IN ('spot','shopping','restaurant','hotel','airport')),
  CONSTRAINT chk_transit_cache_status CHECK (status IN ('ready','stale','failed'))
);

CREATE UNIQUE INDEX IF NOT EXISTS uq_transit_cache_city_from_to
ON transit_cache (city, "fromPointId", "toPointId");
CREATE INDEX IF NOT EXISTS idx_transit_cache_city ON transit_cache (city);
CREATE INDEX IF NOT EXISTS idx_transit_cache_from ON transit_cache ("fromPointId");
CREATE INDEX IF NOT EXISTS idx_transit_cache_to ON transit_cache ("toPointId");
CREATE INDEX IF NOT EXISTS idx_transit_cache_updated_at ON transit_cache ("updatedAt");
```
